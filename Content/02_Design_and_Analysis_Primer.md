# Design $\neq$ Modelling $\neq$ Analysis
Take a moment to think about the terms **Analysis**, **Modelling**, and **Design**. Do you have a clear sense about the difference in these terms and their meaning in the context of aircraft structural engineering? 

Don't beat yourself up if you do not immediately recognize the difference. It is a common misconception among students to view design, modelling, and analysis as interchangeable terms. However, these three stages represent distinct intellectual processes required to develop a safe and efficient airframe. So let's clarify the difference between these terms and how they relate!

## Modelling: *The Act of Simplification*
:::{tip} 📖 Definition
:icon: false
*Modelling* is the process of simplifying a real system using assumptions and abstractions so it can be easily analyzed, while still capturing its key behavior.
:::

## Analysis: *The Execution of Theory*
:::{tip} 📖 Definition
:icon: false
*Analysis* is the process of executing theoretical and mathematical methods on a model to determine how the system responds.
:::

:::{pull-quote}
Analysis answers: “*Given this model, what happens?*”

## Design: *Decisions Supported by Information*
:::{tip} 📖 Definition:
:icon: false
*Design* is the process of making informed decisions to create or improve a system, guided by the information and insight obtained from modelling and analysis.

## A Comparative Framing
To help solidify the differences between Modelling, Analysis, and Design, please have a look at the table below which compares these processes across various aspects. It will be useful in helping you differentiate the various activities you will perform within this course and help place them in the correct context. 

| *Aspect*     | Analysis          | Modelling                 | Design                              |
| ------------ | ----------------- | ------------------------- | ----------------------------------- |
| Core role    | Compute           | Represent                 | Decide                              |
| Input        | Defined model     | Real-world problem        | Requirements & constraints          |
| Output       | Numerical results | Idealized system          | Engineering solution                |
| Skill focus  | Methods & tools   | Assumptions & abstraction | Trade-offs & judgement              |
| Failure mode | Numerical errors  | Wrong assumptions         | Poor decisions despite correct data |


# The University Assessment Conundrum

:::{important} We have done you dirty!
Although *modelling* and *design* are critical skills for any engineer, they add a complication to university assessment - there is not one diffinitive correct answer to modelling or design. As a result, most of your assessments in university are based on *analysis* with pre-defined modelling choices that direct you to an expected outcome. This makes grading the work easier, but can leave you with the sense that all engineering problems have a single *correct* answer. 
:::
